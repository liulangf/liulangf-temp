# 代码可读与性能
在 Python 中有这样一个哲学： Python 代码是为了简单、可读且可维护, 而性能优化的部分则统统应该放在 C++ 中实现.

与之对应的LINQ哲学： 用LINQ 的方法编写所有代码，而将优化的部分统统封装到自定义的查询操作符中。
